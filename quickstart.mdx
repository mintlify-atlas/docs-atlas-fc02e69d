This guide will get you from zero to a running Cookie Party Invitro demo in just a few minutes. You'll clone the repository, install dependencies, and launch the development server.

<Note>
  This quickstart focuses on running the demo. For full development setup including Blender integration, see the [setup guide](/setup).
</Note>

## Prerequisites

You'll need Node.js installed on your system. The demo was built with Node.js, so any recent LTS version should work.

## Clone the repository

First, clone the Cookie Party Invitro repository:

```bash
git clone https://github.com/leon196/CookiePartyInvitro.git
cd CookiePartyInvitro
```

## Install dependencies

Install the required npm packages:

```bash
npm install
```

This installs:
- `concurrently` - Runs multiple npm scripts simultaneously
- `socket.io-client` - WebSocket client for Blender communication
- `http-server` - Local development server
- `chokidar-socket-emitter` - File watching for hot reload

## Run the development server

Start the development server with:

```bash
npm run dev
```

This command runs two processes concurrently:

<CodeGroup>
```bash package.json scripts
"server": "http-server ./demo"
"watch": "chokidar-socket-emitter -p demo"
"dev": "concurrently --kill-others \"npm run server\" \"npm run watch\""
```
</CodeGroup>

The `server` script serves the demo directory on a local HTTP server, while `watch` monitors files for changes and emits socket events for hot reload.

## What to expect

Once the server starts, you should see output like:

```
Starting up http-server, serving ./demo
Available on:
  http://127.0.0.1:8080
  http://192.168.1.x:8080
```

Open your browser and navigate to `http://localhost:8080` to see the demo.

### What you'll see

- A fullscreen WebGL canvas with 3D visualization
- Animated particles and geometry
- Post-processing effects including blur and tinting
- Interactive mouse controls

<Note>
  Without Blender connected, the demo runs with default animations. To control animations from Blender in real-time, follow the [complete setup guide](/setup).
</Note>
